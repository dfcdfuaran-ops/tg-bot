## Production Setup - Summary of Changes

### âœ¨ New Features

#### 1. **Production Docker Compose** (`docker-compose.production.yml`)
- Optimized for server deployment
- Does NOT mount `./src` and `./scripts` directories
- Uses ready-built image from GitHub Container Registry (`ghcr.io/dfteams/remna-tg-bot:latest`)
- Only mounts: `logs/`, `assets/`, `backups/` directories
- Minimal disk footprint on server (~10 MB vs 500+ MB with src)

#### 2. **Enhanced .dockerignore**
- Excludes unnecessary files from Docker image
- Includes: documentation (*.md), IDE files, dev configs, build artifacts
- Result: smaller image size, cleaner deployment

#### 3. **Dual-mode install.sh**
- New `--prod` flag for production installation
- New `-p` shorthand
- Different messages and behavior for dev vs production
- Automatically selects correct `docker-compose` file
- For development: `./install.sh`
- For server: `./install.sh --prod`

#### 4. **Documentation**
- **DEPLOYMENT.md** - Complete DevOps guide
  - Explanation of dev vs production setup
  - How Docker build process works
  - Image pulling and updating
  - Security considerations
  - FAQ section

- **SERVER_QUICK_GUIDE.md** - Quick reference for sysadmins
  - 30-second quick start
  - Essential commands
  - Troubleshooting tips
  - FAQ

#### 5. **Server Setup Script** (`server-setup.sh`)
- Simplified one-command setup for servers
- Wrapper around `install.sh --prod`
